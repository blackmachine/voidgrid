# Void Transfer Protocol (VTP) Specification
**Version:** 1.1 (Draft - Living Document)

## 1. Общие положения
VTP — это компактный бинарный протокол для удаленного управления тайловым терминальным движком VoidGrid. 
Протокол является **двусторонним (Full-Duplex)**. Сервер (микроконтроллер, Python-бэкенд) отправляет команды отрисовки (RX), а Терминал отправляет обратно события ввода (TX).

### 1.1. Типы данных и Endianness
Все числа в протоколе передаются в формате **Little-Endian (LE)**.
* `u8`: 1 байт
* `u16`: 2 байта (LE)
* `u32`: 4 байта (LE)
* `color`: 4 байта (R, G, B, A)
* `string`: строка (`len: u16` + `len` байт UTF-8).

### 1.2. Состояние терминала (Terminal State)
Парсер поддерживает контекст для минимизации трафика:
* `active_buffer` (ID буфера)
* `cursor_x`, `cursor_y` (u32)
* `fg_color`, `bg_color` (color)
* `variant_id` (u8)

## 2. Команды Сервер -> Терминал (Отрисовка и Управление)

### Группа 1: Управление состоянием (State Control)
* **`0x01` SET_BUFFER** `[len: u16] [name: String]` — Сменить активный буфер.
* **`0x02` SET_CURSOR** `[x: u16] [y: u16]` — Переместить курсор.
* **`0x03` SET_FG_COLOR** `[r, g, b, a]` — Установить цвет текста.
* **`0x04` SET_BG_COLOR** `[r, g, b, a]` — Установить цвет фона.
* **`0x05` SET_VARIANT** `[variant: u8]` — Установить индекс варианта шрифта.
* **`0x06` SET_PALETTE_COLOR** `[palette_id: u8] [color_index: u8]` *(Planned)* — Установить цвет из палитры для экономии трафика.

### Группа 2: Вывод данных (Drawing)
* **`0x10` PRINT_CHAR** `[code: u32]` — Напечатать 1 символ и сдвинуть курсор.
* **`0x11` PRINT_STRING** `[len: u16] [text: String]` — Напечатать UTF-8 строку (с обработкой `\n`).

### Группа 3: Продвинутые возможности (Smart Terminal Features)
* **`0x20` LOAD_SCENE_VPK** `[size: u32] [data: Bytes]` *(Planned)* — Загрузка ресурс-пака прямо в оперативную память терминала. Позволяет микроконтроллеру "прошить" UI при подключении.
* **`0x30` RUN_SCRIPT** `[len: u16] [name: String]` *(Planned)* — Запуск локального Lua/Rhai скрипта (анимация, сложный UI) на стороне терминала для экономии сетевого канала.
* **`0x31` STOP_SCRIPT** `[len: u16] [name: String]` *(Planned)* — Остановка работающего скрипта.

---

## 3. Команды Терминал -> Сервер (События ввода)

Эти пакеты генерируются самим движком VoidGrid и отправляются обратно на управляющее устройство.

* **`0x80` MOUSE_CLICK** `[button: u8] [x: u16] [y: u16]` *(Planned)* — Клик мышью в координатах тайловой сетки.
* **`0x81` KEY_PRESS** `[keycode: u32] [modifiers: u8]` *(Planned)* — Нажатие клавиши на клавиатуре.
* **`0x90` EVENT_PAYLOAD** `[len: u16] [data: Bytes]` *(Planned)* — Произвольный ответ от локального Lua-скрипта (например, отправка заполненной формы).

---

## 4. Обработка фрагментации
Пакеты могут приходить частями (TCP/UART). Парсер обязан накапливать байты в буфере и извлекать их только тогда, когда доступна полная длина команды (включая динамические строки и `.vpk` архивы).
