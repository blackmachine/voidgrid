# VoidGrid Engine Roadmap

## ✅ Этап 1: Архитектурный рефакторинг (Завершено)
- [x] Изолированы `AssetManager` и `BufferManager`.
- [x] Настроен God-объект `Grids` и ECS-подобный пайплайн рендера.

## ✅ Этап 2: Scene Manifests & Resource Packs (Завершено)
- [x] Абстракция `ResourceProvider` (Папки / `.vpk` ZIP-архивы).
- [x] Загрузка текстур, шейдеров и JSON из массивов байт в памяти.
- [x] Структура `manifest.json` и `PackLoader` для декларативной сборки сцен.

## 🟡 Этап 3: Void Transfer Protocol (VTP) MVP (В работе)
**Цель:** Базовый бинарный протокол для удаленного управления.
- [x] **State Machine (`VtpState`):** Хранение контекста терминала.
- [x] **Парсер потока байт:** Защита от фрагментации пакетов, чтение Little-Endian.
- [x] **Базовые опкоды:** `SET_BUFFER`, `SET_CURSOR`, цвета, `PRINT_STRING`.
- [x] **Сетевой Транспорт:** Интеграция парсера с реальным TCP-сокетом (или Serial-портом) для приема данных извне, а не из тестового массива.

## 🚀 Этап 4: Smart Terminal & Cargo Workspaces (В планах)
**Цель:** Превратить движок в полноценный "умный дисплей" с возможностью скриптинга и двусторонней связи.
- [x] **Разделение на Workspaces:** Разнести монолит на `voidgrid-engine` (рендерер), `voidgrid-vtp` (протокол) и `voidgrid-term` (приложение).
- [ ] **Client-Side Scripting:** Встраивание Lua (через `mlua`) или `rhai` для выполнения тяжелых анимаций/UI на клиенте.
- [ ] **VTP TX (Transmit):** Реализация отправки пакетов ввода (клики мыши, клавиатура) от терминала обратно к серверу.
- [ ] **VPK Streaming:** Реализация опкода `LOAD_SCENE_VPK` для загрузки целых сцен по сети прямо в память.
